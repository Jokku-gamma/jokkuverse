Project Description: Tic-Tac-Toe Multiplayer Game

This project implements a real-time multiplayer Tic-Tac-Toe game, demonstrating seamless communication between a mobile application and a backend server using WebSockets.

1. Backend (Python with FastAPI and WebSockets)

  • Framework: Built with FastAPI, a modern, fast (high-performance) web framework for building APIs with Python 3.7+ based on standard Python type hints.
  • Real-time Communication: Utilizes WebSockets to enable real-time, bidirectional communication between the server and connected clients (Flutter apps).
  • Game Logic: Manages game state, player connections, turns, move validation, and win/draw conditions. Each game room is isolated, allowing multiple games to run concurrently.
  • Player Management: Handles player joining and leaving rooms, assigning 'X' and 'O' symbols, and synchronizing game state across all players in a room.
  • Deployment: The FastAPI backend is deployed on Render.com, providing a live and accessible WebSocket endpoint.

2. Frontend (Flutter Mobile Application)

  • User Interface: Developed using Flutter, providing a clean, responsive, and intuitive interface for playing Tic-Tac-Toe.
  • WebSocket Integration: Connects to the FastAPI backend via the `web_socket_channel` package, sending player moves and receiving real-time game state updates.
  • Game Flow: Allows users to enter a Room ID and Player ID to join or create a game. Displays the current board, turn, game status (waiting, playing, win/draw), and assigned player symbol.
  • Interactive Board: Users tap on empty cells to make their move, with the UI updating instantly based on server responses.
  • Error Handling: Includes mechanisms for handling disconnections and displaying server-side error messages to the user.
  • Reset Functionality: Provides a button to reset the game, sending a request to the backend.
