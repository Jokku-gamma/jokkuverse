AuthCheckApp: Multi-Stakeholder Login System

This is a dummy Flutter mobile application demonstrating a multi-stakeholder login system. It features three distinct stakeholder roles, each with unique login credentials, and directs them to different pages upon successful authentication. All user data and authentication are managed using Firebase and Firestore.

Project Description: AuthCheckApp

This project is a demonstration of a secure and adaptable authentication system built with Flutter, Firebase, and Firestore, designed to manage multiple user roles or "stakeholders."

1. Flutter Mobile Application

  • Multi-Stakeholder Architecture: The app is structured to differentiate between three distinct user roles: **stakeholder1**, **stakeholder2**, and **stakeholder3**.
  • Firebase Authentication: User logins are securely handled using Firebase Authentication, providing robust and scalable user management. Each stakeholder has a unique email and password for login.
  • Firestore for Data Storage: User data and role-specific information are stored and managed efficiently using Firestore, ensuring real-time data synchronization.
  • Dynamic Page Redirection: Upon successful login, users are automatically directed to specific pages tailored to their respective stakeholder roles, demonstrating role-based access control.
  • Dummy Project: This project serves as a functional example of how to implement multi-role authentication in a Flutter application.

Key Features:

  • Secure user authentication with Firebase.
  • Role-based access control for different user types.
  • Dynamic navigation based on user roles.
  • Firestore integration for flexible data storage.
